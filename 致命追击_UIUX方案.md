# 《致命追击：混沌法则》UI/UX 风格与方案

## 1. 设计定位

### 1.1 核心目标

UI/UX 的核心目标是让玩家在极度混乱的网页端多人乱斗中，始终知道三件事：

1. 我该追谁。
2. 谁正在威胁我。
3. 我的身体和技能还能做什么。

本作的界面不能像传统 RPG 那样堆满面板，也不能像普通射击游戏那样只给准星和血条。它需要同时服务追击链、断肢实战效果、亡者干扰、假动作博弈和荒诞结算。

### 1.2 视觉关键词

- 极简几何
- 高对比血色
- 地下竞技场
- 荒诞暴力喜剧
- 快速、尖锐、低遮挡

### 1.3 设计原则

- 战斗信息优先：真实箭矢、追击目标、肢体状态、危险技能预警拥有最高视觉优先级。
- 装饰瞬时爆发：血浆、断肢、喷涂、涂鸦只在关键时刻爆发，不能长期遮挡核心信息。
- 死亡不是退出：死亡后的 UI 要立刻引导玩家选择寄生或幽灵形态。
- 状态不靠长文：尽量使用图标、轮廓、方向、短标签和动画表达。
- 网页端即开即玩：首屏到匹配的路径必须短，复杂配置放到二级界面。

## 2. 整体视觉风格

### 2.1 美术语言

采用“极简几何血腥终端”风格：

- 角色：火柴人/几何体轮廓，身体部位用独立段落表现，便于断肢状态可视化。
- 地图：低饱和地表，交互元素用材质轮廓区分，如黑墙、木箱、高草、泥沼。
- UI：锋利线条、切角面板、压缩信息条、短促状态标签。
- 特效：真实危险用干净高亮线，血腥反馈用不规则喷溅形状。

### 2.2 色彩系统

推荐色彩分层：

```text
背景底色        炭灰黑 / 油污灰
普通 UI         骨白 / 冷灰
玩家自身        骨白描边 + 阵营色点缀
当前目标        猎杀红
追击我的人      警戒黄
亡者/幽灵       幽蓝白
草丛隐身        脏绿
泥沼减速        淤泥褐
可破坏物        木屑橙
大招危险        高亮橙红
肢体损失        深红 + 缺口图形
```

避免整套界面被单一红黑主宰。红色只用于猎杀、受伤、血浆和高危事件；背景应偏中性，让战斗信号跳出来。

### 2.3 字体与排版

网页端建议：

- 标题字体：窄体、硬朗、压缩感强的 display sans。
- 正文字体：清晰易读的中文黑体或无衬线字体。
- 数字字体：可用轻微机械感的数字样式，但不要全站等宽。

排版原则：

- 局内 UI 使用短词和数字，不使用长句。
- 结算页可以使用更夸张的标题字重。
- 重要战斗状态使用图形优先，文字只做补充。

## 3. 信息优先级

### 3.1 局内信息层级

```text
S 级：真实致命危险
箭矢飞行、弹反窗口、大招路径、毒圈边界、即将命中的流星锤

A 级：玩家决策状态
当前追击目标、被追击警告、肢体损失、翻滚冷却、蓄力状态

B 级：环境与策略信息
草丛隐身、泥沼减速、木箱耐久、空投刷新

C 级：社交与表现
喷涂、表情、击杀文字、血浆、断肢飞散
```

### 3.2 视觉冲突规则

- 当 S 级危险出现时，C 级特效自动降透明度或延迟播放。
- 玩家附近 2 屏范围内的真实箭矢必须永远可见。
- 幽灵假射线必须与真实箭矢预警不同：假射线更虚、更抖、更短暂。
- 喷涂只能污染地图表面，不能覆盖 HUD、目标罗盘和冷却状态。

## 4. 核心界面方案

### 4.1 首屏与主菜单

目标：3 秒内让玩家进入匹配。

布局：

- 中央偏左：游戏标题和当前赛季短标签。
- 中央主操作：`开始追击`。
- 次级操作：`自定义房间`、`外观仓库`、`训练场`。
- 右侧或底部：最近战绩、碎肉代币、每日挑战。

UX 要点：

- 默认焦点落在开始追击。
- 不做长篇世界观介绍。
- 背景可以是实时小型竞技场循环回放，展示箭矢、断肢、幽灵假射线和追击罗盘。

### 4.2 大厅备战

目标：快速确认玩家、选择天赋、生成追击链。

界面模块：

- 玩家列表：头像、昵称、延迟、外观预览。
- 天赋三选一：每张只展示图标、名称、短效果。
- 倒计时：明确进入对局时间。
- 聊天替代：表情和短语按钮，避免打字拖慢节奏。

天赋卡设计：

- 不使用大卡片堆叠风格。
- 使用横向紧凑条目，突出选择状态和快捷键。
- 鼠标悬停显示详细说明，默认只显示核心效果。

### 4.3 局内 HUD

局内 HUD 要极少，但必须稳。

推荐布局：

```text
左上：当前排名 / 击杀分 / 延迟
上中：阶段倒计时 / 毒圈提示
右上：空投与大招状态
左下：肢体状态 + 移动受损提示
下中：翻滚、弹反、当前技能冷却
右下：亡者干扰或社交轮盘入口
角色脚下：追击目标罗盘
准星附近：蓄力、假动作、弹反窗口
```

HUD 规则：

- 所有面板贴边，不浮在战斗中心。
- 冷却用环形或短条，不用长文字。
- 肢体损失 UI 用人体小轮廓表达，断掉的部位直接缺口或闪烁。
- 追击罗盘不显示完整路径，只给方向、距离感和目标强度。

### 4.4 追击链 UI

目标：让玩家知道“我该追谁”，但不知道整条链。

表现方式：

- 角色脚底显示微弱罗盘箭头，指向目标方向。
- 靠近目标时箭头变短、颜色加深、脉冲变快。
- 目标被第三方击杀时，屏幕边缘出现短促红色断链动画。
- 新目标转移到抢怪者时，罗盘重锁并显示 `抢猎者` 标签 1 秒。

避免：

- 不显示完整玩家列表中的追击关系。
- 不用大弹窗打断操作。
- 不用长文本解释规则，结算页再详细回放。

### 4.5 肢体损失 UI

肢体损失有实战效果，因此需要强反馈和持续可读。

局内表现：

- 玩家身体模型直接缺失对应部位。
- HUD 左下角显示简化人体轮廓。
- 新断肢时出现短促红色冲击框，随后收敛到角落状态。

能力影响提示：

```text
断臂：拉弓慢 / 准星抖
双臂断：无法用弓 / 可踢咬
断腿：移速低 / 翻滚短
双腿断：爬行 / 无法翻滚
躯干重伤：持续流血 / 易处决
```

UX 要点：

- 断肢瞬间要爽，持续状态要安静。
- 不要在屏幕中央长期显示受伤说明。
- 能力变化必须能被玩家立刻感知，比如准星抖动、移动拖拽感、冷却图标变形。

### 4.6 蓄力、假动作与弹反 UI

蓄力：

- 准星外圈逐渐闭合或拉伸成箭形。
- 满蓄力时出现短促尖峰，但不持续闪烁。

假动作：

- 取消时播放虚线残影和假拉弓音效。
- 自己视角可见明显取消反馈，敌人视角只看到足以被骗的动作和声音。

弹反：

- 弹反窗口用角色前方短弧线表示。
- 成功弹反触发清脆闪光；失败只给轻微空挥反馈。

### 4.7 亡者界面

死亡后界面不进入传统灰屏旁观，而是立即转为“黄泉选择”。

死亡瞬间流程：

1. 0.3 秒击杀反馈。
2. 屏幕色彩抽离，尸体和击杀者高亮。
3. 出现两项选择：`寄生肉盾`、`幽灵老六`。
4. 3 秒未选择时默认推荐与当前局势更相关的形态。

寄生肉盾 UI：

- 宿主身体旁出现挂载点。
- 擦箭可接时出现极短窗口提示。
- 成功夺舍时用反向血色爆闪，身份 UI 立刻切回生者。

幽灵老六 UI：

- 自身为半透明轮廓，平时 HUD 极简。
- 假射线按钮有冷却与显形风险提示。
- 显形 0.5 秒时边缘出现幽蓝警戒框，提醒自己暴露。

### 4.8 社交与喷涂

目标：允许整活，但不能破坏战斗。

表情轮盘：

- 按键唤出，松开释放。
- 只占屏幕局部，时间冻结不应该存在。
- 表情使用高识别黑白图形加少量色块。

喷涂：

- 可喷在尸体附近、宿主身上或地图表面。
- 战斗中心区域自动降低喷涂透明度。
- 同屏喷涂数量有上限，旧喷涂淡出。

### 4.9 空投与大招 UI

空投：

- 地图边缘出现方向提示和短倒计时。
- 空投落地范围使用危险圈，不用大文本。

空间折叠：

- 锚点成功后显示张力线。
- 张力越高，线越粗、越红、音效越紧。
- 回弹路径必须提前 0.4 到 0.8 秒给出危险预警。

人肉流星锤：

- 链条轨迹显示为粗短拖影。
- 被甩小弟视角需要提示 `千斤坠` 可用窗口。
- 对旁观者和目标展示重击落点预警。

### 4.10 结算页

结算页要强化荒诞叙事，让玩家愿意再来一局。

主要内容：

- 冠军：吃鸡冠军。
- 特别奖：最佳二五仔、究极老六、碎得最均匀、抢猎者之死、假动作大师。
- 追击链回放：用简化环形图展示目标转移和复仇链。
- 肢体战报：展示断肢次数、缺腿存活时间、夺舍后存活时间。
- 奖励：碎肉代币、外观进度、挑战完成。

UX 要点：

- 第一屏先给结果和荒诞称号。
- 详细数据放二级展开。
- `再来一局` 始终为最强操作。

### 4.11 外观仓库与商店

外观只影响表现，不暗示强度。

分类：

- 火柴人皮肤。
- 断肢喷发效果。
- 箭矢拖尾。
- 喷涂。
- 表情包。
- 击杀横幅。

展示方式：

- 提供小型实时预览场景。
- 断肢特效可在假人身上预览。
- 所有外观文案避免 `更强`、`增益`、`稀有=强度`。

## 5. 响应式方案

### 5.1 桌面端

桌面端是首要局内体验：

- 键鼠操作。
- 宽屏 HUD 贴边。
- 鼠标悬停显示高级说明。
- 可支持 16:9、16:10、21:9。

### 5.2 移动端

首版移动端建议定位为非核心对战体验：

- 支持登录、背包、商店、结算、观战。
- 若支持移动局内操作，需要单独设计虚拟摇杆和自动瞄准辅助。
- 不应直接把桌面 HUD 缩小塞进手机屏幕。

### 5.3 小屏适配

- HUD 从四角贴边变成上下两条信息带。
- 肢体状态保留，缩成单个可展开人体图标。
- 结算页优先显示称号和奖励，详细战报纵向滚动。

## 6. 可访问性与可读性

### 6.1 色盲与对比

- 目标、威胁、幽灵、环境状态不能只靠颜色区分。
- 每种状态必须有形状差异：箭头、缺口、虚线、脉冲、边框样式。
- 关键文字与背景保持高对比。

### 6.2 减少动效

提供 `减少血腥动效` 和 `降低屏幕震动` 设置。

- 血浆粒子减少。
- 屏幕震动幅度降低。
- 断肢仍显示状态变化，但减少飞散物。
- 不降低真实危险提示。

### 6.3 音频提示

关键声音需要可区分：

- 真实拉弓。
- 假拉弓。
- 箭矢擦过。
- 弹反成功。
- 幽灵显形。
- 空投落地。

音频 UI 中应允许单独调节：主音量、战斗提示、血腥反馈、表情喷涂。

## 8. 首版 UI/UX 验收标准

MVP 阶段至少满足：

1. 玩家 3 秒内能找到开始匹配入口。
2. 局内不看说明也能知道当前追击方向。
3. 断臂、断腿和双腿断裂的能力变化能被看见、感觉到、解释清楚。
4. 死亡后 3 秒内能选择亡者形态并重新参与。
5. 真实危险提示不会被喷涂、血浆或表情遮挡。
6. 结算页能清楚展示胜者、荒诞称号、追击链变化和奖励。
7. 桌面浏览器 1366x768 下所有关键 HUD 不重叠。
8. 降低动效设置开启后，仍能保留所有关键战斗信息。

## 9. 下一步设计产物

建议下一步按顺序产出：

1. `局内 HUD 线框图`：优先确认追击罗盘、肢体状态、冷却条和危险提示位置。
2. `视觉 token 草案`：颜色、字体、间距、描边、层级、动效时长。
3. `核心组件状态表`：每个 HUD 组件在生者、亡者、断肢、毒圈、大招阶段的状态。
4. `M1 可交互网页原型`：用 PixiJS 做一个带假战斗场景的 HUD 原型。
